## 交互流程

### 流程 1: 查看网络日志

```
1. 长按悬浮按钮
   ↓
2. 显示日志列表
   ↓
3. 点击某个请求
   ↓
4. 显示详情页面
   ↓
5. 切换标签查看不同信息
   ↓
6. 点击关闭返回列表
```

### 流程 2: 搜索和过滤

```
1. 打开日志列表
   ↓
2. 在搜索框输入关键词
   ↓
3. 列表实时过滤
   ↓
4. 选择过滤器（GET/POST/成功/失败）
   ↓
5. 查看过滤后的结果
```

### 流程 3: 导出日志

```
1. 打开日志列表
   ↓
2. 点击"导出"按钮
   ↓
3. 选择分享方式
   ↓
4. 发送 JSON 文件
```

## 实际使用场景

### 场景 1: 调试 API 接口

```
问题：API 返回数据不正确

解决步骤：
1. 长按悬浮按钮打开日志
2. 找到对应的 API 请求
3. 查看"请求Body"确认发送的参数
4. 查看"响应Body"确认返回的数据
5. 查看"状态码"确认请求是否成功
6. 定位问题所在
```

### 场景 2: 检查网络性能

```
问题：页面加载慢

解决步骤：
1. 打开日志列表
2. 查看所有请求的耗时
3. 找出耗时最长的请求
4. 点击查看详情
5. 分析是否需要优化
```

### 场景 3: 排查网络错误

```
问题：某个功能报错

解决步骤：
1. 打开日志列表
2. 使用过滤器选择"失败"
3. 查看所有失败的请求
4. 点击查看详情
5. 在"异常信息"标签查看错误原因
```

### 场景 4: 对比请求参数

```
问题：同一个接口有时成功有时失败

解决步骤：
1. 打开日志列表
2. 搜索该接口的 URL
3. 对比成功和失败的请求
4. 查看"请求Headers"和"请求Body"
5. 找出差异点
```

## 高级技巧

### 技巧 1: 快速定位问题请求

使用过滤器组合：
1. 先选择请求方法（GET/POST）
2. 再选择状态（失败）
3. 使用搜索框输入关键词

### 技巧 2: 导出特定日志

1. 使用过滤器筛选需要的请求
2. 点击导出按钮
3. 导出的 JSON 只包含过滤后的数据

### 技巧 3: 隐藏悬浮按钮

录屏或截图时：
1. 点击悬浮按钮隐藏
2. 完成后重新打开应用
3. 悬浮按钮会自动显示

### 技巧 4: 查看大量数据

响应数据很大时：
1. 在详情页面可以滚动查看
2. 使用等宽字体便于阅读
3. JSON 自动格式化，层次清晰

## 颜色说明

### 请求方法颜色
- 🔵 GET - 蓝色
- 🟢 POST - 绿色
- 🟠 PUT - 橙色
- 🔴 DELETE - 红色
- ⚫ 其他 - 灰色

### 状态码颜色
- 🟢 2xx (成功) - 绿色
- 🟠 3xx (重定向) - 橙色

- 🟣 5xx (服务器错误) - 紫色
- ⚫ 其他 - 灰色

### 耗时颜色
- 🟢 < 500ms - 绿色（快）
- 🟠 500ms - 2s - 橙色（中等）
- 🔴 > 2s - 红色（慢）
