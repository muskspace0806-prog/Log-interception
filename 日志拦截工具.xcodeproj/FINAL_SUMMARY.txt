📦 项目概述
================================================================================
一个功能完整的 iOS 网络请求拦截和日志查看工具，完美复刻 Android 版本功能。

✨ 核心功能
- ✅ 拦截所有 URLSession 网络请求
- ✅ 可拖拽的悬浮按钮（点击隐藏，长按显示）
- ✅ 完整的日志列表和详情页面
- ✅ 搜索、过滤、导出功能
- ✅ JSON 自动格式化
- ✅ 实时显示耗时和状态码

🎯 使用方法
================================================================================

基本操作：
1. 启动应用 → 看到右下角悬浮按钮 📊
2. 拖拽按钮 → 移动到任意位置
3. 点击按钮 → 隐藏按钮
4. 长按按钮 → 显示网络日志列表
5. 点击列表项 → 查看详细信息

日志列表功能：
- 🔍 搜索框：搜索 URL
- 🎛️ 过滤器：按方法或状态过滤
- 🗑️ 清空按钮：清空所有日志
- 📤 导出按钮：导出为 JSON

详情页面（7 个标签）：
1. 基本信息 - 时间、状态码、耗时
2. URL 信息 - 完整 URL 和参数
3. 请求 Headers
4. 请求 Body
5. 响应 Headers
6. 响应 Body（JSON 格式化）
7. 异常信息

================================================================================
⚠️ 注意事项
================================================================================

1. 建议仅在开发/测试环境使用
2. 大量请求会占用内存，定期清空
3. 日志可能包含敏感信息
4. 生产环境使用编译条件控制：
   #if DEBUG
   NetworkInterceptorManager.shared.startIntercepting()
   #endif

================================================================================
❓ 常见问题
================================================================================

Q: 编译错误 "Cannot find type 'NetworkInterceptor'"
A: 检查文件是否添加到 Target

Q: 看不到悬浮按钮
A: 检查 SceneDelegate 是否正确更新

Q: 拦截不到请求
A: 检查 AppDelegate 是否启动拦截

Q: 如何在生产环境禁用？
A: 使用 #if DEBUG 编译条件

更多问题请查看：INSTALLATION_GUIDE.md
